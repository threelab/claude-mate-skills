## 做什么样的界面？

我们用**网页**来做界面。为什么？

| 方式 | 优点 | 缺点 |
|------|------|------|
| 网页（HTML） | 简单、跨平台、用户熟悉 | 需要一个本地服务器 |
| 桌面应用（Electron） | 体验好 | 开发复杂、体积大 |
| 命令行（CLI） | 最轻量 | 用户不会用 |

我们选**网页**的方式：
- 用户双击一个启动脚本
- 自动打开浏览器显示界面
- 用户在网页里操作
- 网页和本地程序通信，完成任务

## 技术怎么实现？

### 架构概述

```
用户的浏览器（前端）
    ↓↑ HTTP 请求
本地 Node.js 服务器（后端）
    ↓
调用工具（FFmpeg、ASR）
```

**前端做什么？**
- 显示界面给用户看
- 用户点击按钮后告诉后端"开始做事"
- 显示进度和日志

**后端做什么？**
- 接收前端的请求（"帮我处理这个视频"）
- 调用工具（FFmpeg 提取音频、语音识别）
- 返回处理结果给前端

## 实际代码：后端服务器

```javascript
// gui/server.js

const http = require('http');
const fs = require('fs');
const path = require('path');
const { execSync, spawnSync } = require('child_process');

// 端口配置
const PORT = 3005;

// 创建服务器
const server = http.createServer((req, res) => {
  // 允许跨域（浏览器安全策略）
  res.setHeader('Access-Control-Allow-Origin', '*');
  res.setHeader('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type');

  // 解析请求
  const url = new URL(req.url, `http://localhost:${PORT}`);

  // 路由分发：根据请求的路径做不同的事
  if (url.pathname === '/' || url.pathname === '/index.html') {
    // 1. 主页 - 返回 HTML 文件
    serveFile(res, path.join(__dirname, 'index.html'));

  } else if (url.pathname === '/api/browse' && req.method === 'POST') {
    // 2. 选择文件 - 弹出文件对话框
    const browseResult = spawnSync('python', [path.join(__dirname, 'browse.py')], {
      encoding: 'utf8',
      windowsHide: true
    });
    const filePath = browseResult.stdout.trim();

    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ path: filePath }));

  } else if (url.pathname === '/api/transcribe' && req.method === 'POST') {
    // 3. 处理视频 - 核心功能
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', () => {
      const data = JSON.parse(body);
      const videoPath = data.videoPath;

      try {
        // 这里调用我们之前写的工具
        // 1. 提取音频
        // 2. 语音识别
        // 3. 保存结果

        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ success: true, message: '处理完成' }));
      } catch (error) {
        res.writeHead(500, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ success: false, error: error.message }));
      }
    });

  } else {
    // 404 - 找不到页面
    res.writeHead(404);
    res.end('Not Found');
  }
});

// 辅助函数：返回文件
function serveFile(res, filePath) {
  if (fs.existsSync(filePath)) {
    const ext = path.extname(filePath);
    const types = {
      '.html': 'text/html',
      '.js': 'application/javascript',
      '.css': 'text/css'
    };
    const contentType = types[ext] || 'application/octet-stream';

    res.writeHead(200, { 'Content-Type': contentType });
    fs.createReadStream(filePath).pipe(res);
  } else {
    res.writeHead(404);
    res.end('File not found');
  }
}

// 启动服务器
server.listen(PORT, () => {
  console.log(`服务器已启动: http://localhost:${PORT}`);
});
```

## 文件选择：怎么让用户选视频？

这里有个问题：网页浏览器出于安全考虑，**不能直接读取本地文件的完整路径**。

我们用一个**Python 小脚本**来解决这个问题：

```python
# gui/browse.py

import tkinter as tk
from tkinter import filedialog

# 创建一个隐藏的窗口
root = tk.Tk()
root.withdraw()

# 弹出文件选择对话框
file_path = filedialog.askopenfilename(
    title='请选择要剪辑的视频文件',
    filetypes=[
        ('视频文件', '*.mp4 *.mov *.avi *.mkv'),
        ('所有文件', '*.*')
    ]
)

# 打印结果（Node.js 程序会读取这个输出）
if file_path:
    # 把斜杠换成 Windows 风格的反斜杠
    print(file_path.replace('/', '\\'))
else:
    print('')
```

**工作原理：**

1. 用户在网页点"浏览"按钮
2. 网页告诉后端："帮我弹一个文件选择框"
3. 后端调用 Python 脚本（browse.py）
4. Python 弹出一个系统的文件选择框
5. 用户选择文件后，Python 把文件路径打印出来
6. 后端拿到路径，返回给网页
7. 网页显示路径给用户看

## 用一个生活类比理解前后端交互

想象你在**银行**办业务：

| 银行流程 | 对应我们的项目 |
|---------|----------------|
| 🏦 银行大厅 | 网页界面（用户看到的） |
| 👩‍💼 柜台服务员 | 后端服务器（接收请求） |
| 📋 业务单据 | HTTP 请求（用户操作） |
| 💰 金库 | 工具和数据（FFmpeg、ASR） |
| 📝 收据 | HTTP 响应（处理结果） |

**流程1：用户存钱（对应：选择视频文件）**

1. 你走进银行（打开网页）
2. 你说"我要开户"（点击"浏览"按钮）
3. 服务员拿起单据（发送 HTTP 请求）
4. 服务员去后台处理（调用 browse.py）
5. 服务员给你回单（返回文件路径）
6. 你看到账号信息（网页显示路径）

**流程2：用户转账（对应：开始处理视频）**

1. 你说"帮我转账100元"（点击"开始处理"）
2. 服务员核对信息（检查视频路径）
3. 服务员去后台操作（调用 FFmpeg、ASR）
4. 后台处理完成（提取音频、语音识别完成）
5. 服务员告诉你"完成了"（返回成功响应）
6. 你收到通知（网页显示"处理完成"）

## 核心概念：理解前后端分离

### 什么是前端？

**前端 = 用户看到的东西**

- 网页长什么样（HTML）
- 样式好不好看（CSS）
- 按钮点了会做什么（JavaScript）
- 给用户的反馈（进度条、日志）

### 什么是后端？

**后端 = 在背后干活的程序**

- 接收用户的请求（"帮我处理视频"）
- 调用工具（FFmpeg、语音识别）
- 处理数据（分析内容、生成报告）
- 返回结果给前端

### 为什么要分离？

| 优点 | 说明 |
|------|------|
| 职责清晰 | 前端管显示，后端管干活 |
| 容易维护 | 改界面不会影响业务逻辑 |
| 用户体验好 | 前端可以快速响应，后端在后台慢慢处理 |
| 扩展性好 | 以后可以做手机 App、桌面客户端，共用同一套后端 |

## 你可能会问的问题

**Q: 为什么要用网页做界面？不能做个桌面程序吗？**

A: 可以做，但网页最简单。HTML/CSS/JavaScript 人人都懂一点，跨平台，不需要编译。如果以后想要桌面程序，可以把网页包进 Electron。

**Q: 启动服务器后，浏览器不会自动打开怎么办？**

A: 在启动脚本里加一行命令（Windows）：`start http://localhost:3005`，或者让用户自己打开浏览器输入地址。

**Q: 为什么需要 Python 来选文件？不能直接用 Node.js 吗？**

A: 可以，但 Python 的 tkinter 库几乎每台电脑都自带，做文件选择对话框最简单。用 Node.js 也能做，但需要额外安装库。
