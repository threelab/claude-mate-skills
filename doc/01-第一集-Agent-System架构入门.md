## 先讲个小故事

假设你是一个**口播博主**，每天要录视频。每次录完，你都要做一件很烦的事——**剪辑**。

比如你录了一段视频，内容是这样的：

> "嗯...那个...今天想跟大家聊一聊...啊...如何提高学习效率。嗯...首先呢...那个...我们要定目标。哦不对，应该先定方向。对，先定方向，然后才是目标。"

你自己听的时候觉得很流畅。但录完回头看视频会发现：

- 开头有很多"嗯、啊、那个、然后"
- 中间说错了，纠正了一遍（"哦不对..."）
- 每句话之间有长时间的停顿

这些内容如果不删掉，视频会很拖沓，观众可能看一半就走了。

**但手动剪辑真的很烦！** 你要：
1. 打开视频软件
2. 一段一段听
3. 找到"废话"的位置
4. 剪掉它
5. 重复以上步骤...

10分钟的视频可能要花20分钟去剪。

---

## 所以我们要做的事

我们的目标很简单：**做一个能自动帮你找到"废话"的工具**。

这个工具会：

1. **听你的视频** - 把视频里的声音提取出来
2. **转换成文字** - 告诉你每个字在第几秒说的
3. **标出来"废话"** - 把语气词、停顿、重复的地方标出来
4. **给你一个界面** - 让你看着文字，点一下就能预览
5. **让你做选择** - 你决定删哪些，工具帮你记下来

就这么简单！

## 我们的项目长什么样？

就像你家里的房间布局一样，我们的项目也需要"分房间"：

```
clipAnnouncement/          ← 这是你的"家"（项目根目录）
│
├── SKILL.md              ← 家规写在这里（告诉别人这个家是做什么的）
│
├── config/               ← 储物间（放配置文件）
│   ├── preferences/       │
│   │   └── user.json     ← 用户的喜好设置（比如喜欢什么剪辑风格）
│   └── principles/        │
│       ├── 1-core.md     ← 核心家规（最重要的规则）
│       ├── 2-fillers.md  ← 语气词规则
│       └── ...           ← 其他规则
│
├── tools/                ← 工具箱（各种工具）
│   ├── asr_engine.js     ← 语音识别工具（听音频、写文字）
│   └── ffmpeg_helper.js  ← 视频处理工具（切视频）
│
├── gui/                  ← 客厅（用户来这里办事）
│   ├── index.html        ← 客厅的装修
│   ├── server.js         ← 管家（帮用户跑腿）
│   └── start_gui.bat     ← 开门的钥匙（启动工具）
│
├── scripts/              ← 厨房（在这里"做菜"）
│   ├── generate_review.js    ← 做一个审核页面
│   └── generate_subtitles.js ← 做字幕
│
└── output/               ← 餐厅（做好的视频放这里）
    └── 日期_视频名/
        ├── 1_转录/       ← 第一步处理的结果
        ├── 2_分析/       ← 第二步处理的结果
        └── 3_审核/       ← 用户审核的结果
```

**为什么要分这么多"房间"？**

想象一下：
- 如果你把做饭的灶台放在客厅（把界面代码和工具代码混在一起），客人来了会很奇怪
- 如果你把厨房和储物间放一起（把配置文件和工具代码混在一起），找东西会很麻烦
- 如果你没有餐厅（没有输出目录），做好的东西不知道放哪里

**分清楚"房间"的好处：**
✅ 找东西方便（想改规则就去 config，想改工具就去 tools）
✅ 其他人来了也能看懂（别人看你的项目也知道什么东西在哪里）
✅ 出问题容易排查（比如"审核页面出错了"→ 直接去 scripts 找）

## Agent System 是什么？

这个听起来很玄乎，其实很简单。

**场景1：你去餐厅吃饭**

你跟服务员说："来一碗番茄鸡蛋面。"

- 服务员（Agent）听懂了你的需求
- 他去后厨告诉厨师要做什么（调用工具）
- 厨师做好后端出来（返回结果）

**场景2：你用我们的剪辑工具**

你跟 AI 说："帮我剪一下这个视频。"

- AI 听懂了你的需求（要剪辑视频）
- 它去调用我们的"剪辑工具"（Skill）
- 工具开始工作：提取音频 → 转写文字 → 分析内容 → 生成结果

**Agent System 就是这个"服务员"的能力**。它能：
1. 听懂你说的话（理解用户意图）
2. 知道该用什么工具（匹配 Skill）
3. 把事情办好（调用工具完成任务）

**我们要做的 Skill 是什么？**

Skill 就是一份说明书，告诉 Agent：

> "如果用户说'帮我剪视频'，你就用我这个工具。具体做法是...（以下是使用说明）"

所以我们的 SKILL.md 就是这份说明书。

## 第一个文件：SKILL.md

这个文件就像产品的使用说明书。我们来写一份：

```markdown
# 口播视频智能剪辑 (clipAnnouncement)

## 这个工具是做什么的？

帮你快速剪辑口播视频，自动识别语气词、停顿和重复内容。

## 什么时候会用到它？

当你说类似以下的话时，这个工具就会被启动：
- "帮我剪一下这个视频"
- "这个口播里的'嗯啊'太多了，帮我处理一下"
- "我想精简这段视频"
- "帮我找出口播里的废话"

## 它能做什么？

✅ 从视频文件里提取音频
✅ 把语音转换成文字（还能告诉你每个字在第几秒）
✅ 识别语气词（嗯、啊、那个、然后...）
✅ 识别长停顿（中间停了几秒没说话）
✅ 识别口误和重复表达
✅ 生成一个审核页面，让你看着文字选择删什么
✅ 帮你记录要删除的片段位置

## 它不能做什么？

❌ 不能自动帮你做最终剪辑（还是需要你确认一下，避免误删重要内容）
❌ 不能理解视频画面（只能处理声音部分）
❌ 不能识别外语（目前只支持中文）
❌ 不能处理直播流（只能处理已录好的视频文件）

## 使用流程

1. 用户选择一个视频文件
2. 工具提取视频里的音频
3. 把音频送去做语音识别
4. 分析识别出来的文字，标记可能需要删除的内容
5. 生成一个网页界面，让用户查看和选择
6. 用户确认后，记录要删除的片段位置

## 输出结果

处理完后，你会得到：
- 一个音频文件（audio.mp3）
- 一份时间戳（每个字在第几秒）
- 一个分析结果（AI 建议删除的片段）
- 一个审核页面（在浏览器里查看和操作）

## 注意事项

⚠️ 第一次使用需要配置 API Key（后面会教）
⚠️ API 调用会产生费用（但很便宜，几毛钱就能处理很长的视频）
⚠️ 最终剪辑请人工确认，AI 的判断仅供参考
⚠️ 建议视频时长不超过 30 分钟
```

## 小结：这一步我们做了什么？

**核心成果：**

1. ✅ 搞清楚了目标（做一个口播剪辑工具）
2. ✅ 规划了项目结构（分"房间"）
3. ✅ 写好了使用说明书（SKILL.md）

**下一步要做什么？**

下一步我们要**定规则**——告诉程序什么样的内容该删，什么样的内容不能删。

就像你招聘一个剪辑助理，不能只说"帮我剪辑"，你得告诉他：

> "嗯、啊、那个"这样的词可以删；
> 中间停顿超过2秒可以删；
> 口误纠正后，保留后面那句正确的就行。

这些规则，就是我们下一步要做的事。
