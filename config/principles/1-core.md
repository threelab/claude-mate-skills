# 核心原则

## 🚨 删前保后（最高优先级）

**此原则优先于所有子规则。任何删除操作都必须先过这一关。**

口播过程中，后说的通常更完整、更准确，所以：
- 删除**前面的**（口误/残句/重复的第一次）
- 保留**后面的**（完整句/纠正后的/重复的最后一次）

### ❌ 常见错误

```
句子: "于是很把于是很容易把它理解成一种不够友好但很高效的界面"
                ↑前半(错)    ↑后半(对,更完整)

❌ 整句删除 → 把后面完整的也删了
✓ 只删"于是很把"这几个字，保留"于是很容易把它理解成..."
```

```
相邻句:
  前句: "到底是什么原因导致了他会在AI的时代重新被选中"
  后句: "到底是什么原因导致了他在AI的时代又重新被选中了呢"

❌ 删后句（更长更完整的）
✓ 删前句，保留后句
```

### 适用范围

| 场景 | 操作 | 说明 |
|------|------|------|
| 重复句 | 删前句，保留后句 | 不是删较短的，是删先说的 |
| 句内重复 | 只删重复片段 | 不删整句，保留完整语义 |
| 残句 | 删残句 | 话说一半被打断，保留后面重说的完整版 |
| 重说纠正 | 删前面说错的 | 保留后面纠正的内容 |
| 口误修正 | 删错误版本 | 保留"不对，应该是..."后面的正确内容 |

### 校验步骤

标记完所有删除项后，逐条检查：
1. 被删的是不是**前面/较早**的版本？
2. 保留的是不是**后面/更完整**的版本？
3. 句内重复是不是**只删了重复片段**，没有误删整句？
4. 删除后句子是否**语法完整、语义连贯**？
5. 删除后朗读是否**自然流畅**？

## 与其他规则的关系

核心原则是所有子规则的**决策依据**：

- **语气词处理**（2-fillers.md）：删除不影响语义的填充词
- **停顿处理**（3-pauses.md）：>2s 静音视为废片删除
- **纠错判定**（4-corrections.md）：删错误版本，保留纠正后的
- **重复检测**（5-logic.md）：删前保后，保留最终版

## 例外

用户明确说"前面更顺"或指定保留前面版本时，删后保前。
除此之外，一律删前保后。
